## 1. 产品概述

### 1.1 产品定位

本产品面向「数字化招聘」场景，在 B/S 架构下提供 **文档上传与解析进度跟踪**、**个人/职位能力图谱可视化**、**双向智能匹配与可解释结果展示**，并配套 **平台管理与运营监控** 界面。项目对齐第十五届中国软件杯大赛 **B2** 赛题《基于银河麒麟操作系统的 AI 智能匹配与能力图谱系统》的能力描述与提交物要求。

### 1.2 核心价值

- **对个人用户**：上传简历类文档，查看解析结构化结果与个人能力图谱，获取职位推荐及匹配依据说明。  
- **对企业用户**：上传岗位说明（JD）类文档，查看解析结果与岗位能力图谱，获取候选人推荐与匹配解释。  
- **对平台运营方**：通过管理端查看用户、文档、匹配与监控类页面，支撑演示与后续扩展为真实运营后台。  

### 1.3 当前仓库中的交付形态

代码与文档按目录拆分：**`frontend/`（前端）**、**`backend/`（后端）**、**`docs/`（文档）**，仓库根目录见 **`README.md`**。

| 组成部分 | 说明 |
|----------|------|
| **Web 前端** | `frontend/`：Vue 3 + Vite + TypeScript + Element Plus + Pinia + Vue Router；图谱 AntV G6；图表 ECharts。 |
| **后端服务** | `backend/`：Spring Boot（JDK 17+），提供 `/api/v1/*` REST 接口；文档上传后结合 **火山引擎 Ark** 大模型 API 做简要分析；业务数据通过 **JPA + Flyway** 持久化至 **GreatSQL**（或与 MySQL 8 协议兼容的实例）。 |
| **设计文档** | `docs/AI智能匹配与能力图谱系统-前端架构设计方案-V2.0.md`：赛题对标、页面与模块、RBAC、图谱与匹配 UI 方案、信创与部署建议等。 |

> **说明**：赛题要求中的「LoongArch + 银河麒麟 V11 部署」等属于**整体系统级**部署与取证要求；数据库侧已可按 **`backend/README.md`** 接入 GreatSQL，信创目标机浏览器与显卡验证需在目标环境单独执行。

### 1.4 完整训练与公开数据基准：Job-SDF

与本仓库当前「演示联调 + Ark 辅助解析 + 库内种子职位 / 规则匹配」不同，凡需**完整、可论文复现的岗位技能需求预测训练与多粒度基准评测**，应以 **Job-SDF** 为指定公开数据集与说明入口（作者单位含**中国科学技术大学**等，详见官网）：

- **官方网站**：[https://job-sdf.github.io/](https://job-sdf.github.io/)  
- **定位**：多粒度岗位技能需求预测（occupation / company / region 等）的数据集与 Benchmark；官网摘要基于国内主要招聘平台 **2021–2023** 海量公开岗位广告，含技能类型月度需求、企业维度等，用于模型训练与评测（具体字段与许可以官方发布为准）。  
- **与本产品关系**：当前交付**不捆绑** Job-SDF 下载与训练代码；若产品化引入市场级预测或再训练匹配模型，须在工程中单独立项（合规、存储、流水线），并**回写**本说明书与 **`软件功能与系统结构说明.md`**。

## 2. 用户角色与功能清单

系统采用 **RBAC 思路**：登录后按 `userType` 分流 **个人（PERSON）**、**企业（COMPANY）**、**管理员（ADMIN）** 三套菜单与路由。

### 2.1 个人端（PERSON）

| 功能模块 | 用户可见能力 |
|----------|----------------|
| 工作台 | 入口聚合、快捷导航至文档与匹配。 |
| 文档中心 | 文档列表相关页面（与后端联调时可展示上传产生的任务）。 |
| 简历上传 | 支持 **PDF / DOC / DOCX** 上传（后端校验）；单文件大小上限与 Spring 配置一致（默认 **50MB**）。 |
| 解析任务 | 按 `docId` 轮询解析状态（后端模拟 **PENDING → PROCESSING → DONE** 时间线）。 |
| 解析结果 | 展示结构化字段 Tab（技能、教育、项目等）及证据说明占位。 |
| 能力图谱 | 基于后端返回的节点/边数据，使用 G6 做交互式图谱（缩放、拖拽等，以实际浏览器与显卡能力为准）。 |
| 职位推荐 | 推荐列表与匹配详情页（评分、分项与可视化以接口与前端实现为准）。 |

### 2.2 企业端（COMPANY）

与个人端对称：**JD 上传**、解析任务/结果、**岗位能力图谱**、**候选人推荐**及匹配详情。

### 2.3 管理端（ADMIN）

| 功能模块 | 用户可见能力 |
|----------|----------------|
| 概览 | 管理后台首页。 |
| 用户管理 | 用户列表与维护类界面。 |
| 文档库 | 文档维度管理入口。 |
| 数据维护 | 技能库/同义词等数据维护入口（具体字段以后端为准）。 |
| 匹配记录 | 匹配任务或记录列表。 |
| 运营监控 | 监控看板类页面。 |
| 日志审计 | 审计与日志查询类页面。 |

## 3. 技术架构摘要

### 3.1 前端

- **运行时**：现代浏览器（信创浏览器需按设计文档做 WebGL/Canvas 与构建目标验证）。  
- **构建**：`npm run dev` 开发；`npm run build` 产出静态资源至 `dist/`，可由 Nginx 托管。  

### 3.2 后端

- **框架**：Spring Boot，默认端口 **8080**。  
- **大模型**：通过 `application.yml` 中 `ark.*` 与环境变量 `ARK_API_KEY`（可选 `ARK_BASE_URL`、`ARK_MODEL`）调用云端模型；文档上传时先用 **Apache Tika** 抽取 PDF/DOC/DOCX 正文（可配置 `aimap.document.*` 长度上限），再将正文与文件名一并送入模型生成结构化点评与霍兰德画像。  
- **数据层与用户认证**：GreatSQL（MySQL 协议）持久化 **`aimap_user`**（`account` 用户名、`phone`/`email` 可选、密码新用户为 **BCrypt**、种子账号兼容明文）、文档任务、人才市场/人才库等；配置见 `application.yml` 与 `DB_*`。**`POST /api/v1/auth/register`** 插入用户，**`POST /api/v1/auth/login`** 校验密码并签发 **`Authorization: Bearer token-{userType小写}.{Base64URL(UTF-8 用户名)}`**（点号分隔，避免 HTTP 头非 ASCII；旧版 **`token-person-demo`** 仍兼容）；文档上传等写操作须携带该头，后端才能把文档绑定到当前用户（人才库依赖此绑定）。

### 3.3 主要 REST 接口（与 `backend/README.md` 一致）

- 认证：`POST /api/v1/auth/register`、`login`、`logout`  
- 文档：`POST /api/v1/document/upload`，`GET .../document/{id}/status`，`GET .../document/{id}/result`  
- 图谱：`GET /api/v1/graph/person/{id}`，`GET /api/v1/graph/job/{id}`，`POST /api/v1/graph/expand`  
- 匹配：`POST /api/v1/match/recommend-jobs`，`POST /api/v1/match/recommend-candidates`，`GET /api/v1/match/{id}/detail`  
- 大模型：`POST /api/v1/llm/extract`、`explain-match`、`generate-suggestion`

## 4. 您现在可以做什么

以下操作均基于当前仓库已实现能力，便于 **本地演示、联调、答辩录屏**。

1. **启动后端**  
   - 进入 **`backend/`**，配置 JDK 17+、Maven，并完成 GreatSQL 建库与 `DB_*`（详见 **`backend/README.md`**）。  
   - 配置 `ARK_API_KEY`（推荐使用 **`backend/.env.backend`**）后执行 `mvn spring-boot:run` 或使用 **`backend/start-backend.ps1`**（无 Key 时 LLM 相关能力可能降级）。  
   - 服务地址：`http://localhost:8080`。  

2. **启动前端**  
   - 在 **`frontend/`** 执行 `npm install`、`npm run dev`，按 `vite` 与 `.env*` 将 `/api` 代理到后端。  

3. **体验完整业务动线**  
   - 使用内置演示账号或自行注册：  
     - 个人：`demo` / `123456`（用户类型选个人）  
     - 企业：`demo` / `123456`（用户类型选企业）  
     - 管理员：`admin` / `123456`  
   - 完成：**登录 → 上传文档 → 查看解析进度与结果 → 打开能力图谱 → 查看推荐列表与匹配详情 → 进入管理端各菜单**。  

4. **对照赛题与评委关注点整理材料**  
   - 直接引用或拆分 **`docs/AI智能匹配与能力图谱系统-前端架构设计方案-V2.0.md`** 中的赛题对标表、测试与验收清单、部署与信创章节，拼入「需求/设计/测试报告/部署说明」。  

5. **工程化与质量**  
   - 执行 `npm run check`、`npm run lint`、`npm run test`（以 `package.json` 脚本为准）做类型检查、规范与单元测试。

## 5. 运行环境与部署要求（摘要）

| 项目 | 建议 |
|------|------|
| 前端 Node | 与 Vite 5、Vue 3.4 生态兼容的 LTS 版本 |
| 后端 JDK | 17 及以上 |
| 构建工具 | Maven 3.9+ |
| 反向代理 | 生产环境建议 Nginx 托管 `dist/` 并反向代理 `/api` 至 Spring Boot |
| 信创目标机 | LoongArch + 银河麒麟高级服务器版 V10/V11（需单独验证浏览器与 WebGL） |
