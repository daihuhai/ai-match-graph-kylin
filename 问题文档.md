### [2026-06-04] 登录时报错 `Request failed with status code 500`

| 项 | 内容 |
|----|------|
| **现象** | 前端登录页点击「登录」后，提示 `Request failed with status code 500`；浏览器网络面板中 `/api/v1/auth/login` 返回 HTTP 500。 |
| **影响范围** | 所有角色登录（个人 / 企业 / 管理员）；开发环境 Vite + Spring Boot 联调。 |
| **环境** | Windows；前端 `npm run dev`（5173）；后端 `start-backend.ps1`；数据库 Docker MySQL（3307）。 |

#### 原因分析

本次 500 **并非**后端业务逻辑返回的「账号密码错误」（那种情况 HTTP 仍为 200，`body.code` 为 400）。实际原因有两类：

1. **Vite 开发代理连不上后端（主要原因）**  
   - 前端通过 Vite 将 `/api` 代理到后端。若后端尚未启动完成，或代理目标端口错误，会出现 `ECONNREFUSED`。  
   - Vite 会将此类代理失败以 **HTTP 500** 返回给浏览器，axios 表现为 `Request failed with status code 500`。  
   - 典型场景：先启动前端、后启动后端，在后端日志出现 `Started AimapBackendApplication` 之前点击登录。

2. **前后端端口配置不一致（次要原因）**  
   - `application.yml` 默认端口为 **8080**。  
   - 曾在 `backend/.env.backend.local` 中配置 `SERVER_PORT=8081`，而 `frontend/vite.config.ts` 代理默认一度为 **8081**，与 README 描述的 8080 不一致。  
   - 若只启动后端且未加载本地 env（走 8080），而 Vite 仍代理到 8081，同样会连接失败并表现为 500。

**补充说明**：使用种子账号 `demo` / `123456` 直接请求后端接口时，登录可正常返回 `code: 200`，说明数据库连接与用户数据无问题。

#### 解决方案

1. **统一开发端口为 8080**  
   - 移除 `backend/.env.backend.local` 中的 `SERVER_PORT=8081`，后端使用默认 8080。  
   - 将 `frontend/vite.config.ts` 中代理默认目标改为 `http://127.0.0.1:8080`（与 README、`application.yml` 一致）。

2. **改进前端错误提示**（`frontend/src/api/http.ts`）  
   - 对无响应、HTTP 500/502/503 等情况，提示「后端不可用 / 可能尚未启动完成」，避免仅显示 axios 的 `status code 500`。

3. **启动顺序建议**  
   1. `docker compose up -d`（数据库）  
   2. `backend/start-backend.ps1`（等待启动完成）  
   3. `frontend` 目录下 `npm run dev`  

#### 验证方式

```powershell
# 1. 确认后端已监听 8080
Test-NetConnection 127.0.0.1 -Port 8080

# 2. 直接调登录接口（应返回 code 200）
Invoke-RestMethod -Uri "http://127.0.0.1:8080/api/v1/auth/login" -Method POST `
  -ContentType "application/json" `
  -Body '{"account":"demo","password":"123456","userType":"PERSON"}'

# 3. 经 Vite 代理（前端 dev 启动后）
Invoke-RestMethod -Uri "http://localhost:5173/api/v1/auth/login" -Method POST `
  -ContentType "application/json" `
  -Body '{"account":"demo","password":"123456","userType":"PERSON"}'
```

#### 涉及文件

| 文件 | 变更说明 |
|------|----------|
| `frontend/vite.config.ts` | 代理默认目标 `8081` → `8080` |
| `frontend/src/api/http.ts` | 连接失败 / 5xx 时友好错误文案 |
| `backend/.env.backend.local` | 移除 `SERVER_PORT=8081` |
